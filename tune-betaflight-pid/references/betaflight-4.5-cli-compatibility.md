# Betaflight 4.5 CLI compatibility

Use this reference when the analyzer emits or reviews a CLI stage for Betaflight 4.5. It is deliberately limited to the variables generated by this Skill; use `get <name>` on the connected flight controller before applying a stage to a different firmware family or custom build.

- [Betaflight 4.5 CLI command reference](https://betaflight.com/docs/wiki/guides/current/Betaflight-4.5-CLI-commands)

| `debug_mode` | includes `RPM_FILTER` | Do not enable automatically or leave it persistent. Use `RPM_FILTER` only for a temporary diagnostic log and restore `NONE` afterward. |

RPM filtering requires a compatible ESC, bidirectional DShot, and valid per-motor telemetry. The official guide recommends a power cycle followed by the Motors-tab RPM/error check before flight. The generated stage does not change the existing DShot protocol because it is emitted only if the log already reports a valid DShot protocol.

### Dynamic damping naming

In Betaflight 4.5, `d_roll` and `d_pitch` are the D Max values, while `d_min_roll` and `d_min_pitch` are base D values. The analyzer records and reduces both together only in its bounded noise-reduction stage; it never raises either automatically.

## CLI application rule

`save` writes settings and reboots the flight controller in Betaflight 4.5. Always capture `diff all`, remove propellers for configuration work, reboot, and then validate in a short prop-on flight with motor-temperature checks.
